The PlusCal Translator

PlusCal is an algorithm language based on TLA+.  A PlusCal algorithm is written as a comment in a TLA+ module.  The PlusCal translator writes the TLA+ translation of the algorithm into the module.  See the PlusCal web page for more information about PlusCal, including a language manual.  An overview of the language is provided by the paper The PlusCal Algorithm Language.

Toolbox support for the PlusCal translator is still fairly primitive.  You run the translator on the module in the currently selected module editor by clicking on  File/Translate PlusCal Algorithm.  Translation errors are displayed in the same  Parsing Error  view where TLA+ parsing errors are displayed.  Some errors in a PlusCal algorithm are not found by the translator, but instead produce parsing errors in the TLA+ module.  Those are displayed as ordinary TLA+ parsing errors.

Currently, the only way to control the translator options is to state explicitly the options with which the translator is called.  This is done by opening the  Spec Explorer  view (see the Manipulating Specs help page), right-clicking on the spec, choosing  Properties, and entering the desired options in the  PlusCal call arguments  field.  For example, to specify weak fairness, you would enter  -wf .

The Toolbox now provides only one PlusCal translator preference.  You can have the translator automatically called when you modify and then save a module containing a PlusCal algorithm.  (More precisely, it is called if the module contains the text   --algorithm  anywhere within it.)  You can set it by selecting the  PlusCal Translator Preferences  page from among the TLA+ Preferences.  (See the Preferences help page.) The default is not to translate on save.
